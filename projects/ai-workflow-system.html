AI SaaS Workflow Automation System

Overview

Automated the end-to-end reservation sync and routing process between database records, cloud applications, and Restaurant Management Systems (RMS) like OpenTable and SevenRooms—reducing manual entry and minimizing booking errors.

The Problem

When new reservations came in, operational teams had to manually input booking details into different RMS platforms (OpenTable or SevenRooms). This manual process was time-consuming, repetitive, and prone to human error, leading to delays and potential booking mix-ups.

The Solution

Designed and deployed an automated, AI-driven workflow that seamlessly captures, formats, processes, and tracks reservations across platforms in real time.

System Architecture

[Airtable Database] 
       │
       ▼ (Zapier Trigger)
[JS Data Formatting]
       │
       ▼ (API)
[Skyvern AI Web Automation] ────► [OpenTable / SevenRooms]
       │
       ▼ (Status: Success / Failure)
[Zapier Webhook Routing]
       ├──► [Slack Alerts (Success & Failures)]
       └──► [Airtable (Status Tracking Tables)]

Key System Architecture & Components

  • Trigger & Orchestration (Airtable & Zapier):
    • Monitors Airtable for incoming reservations from the main database.
    • Triggers automated "Zaps" immediately upon receiving new entries.
  • Custom Data Transformation:
    • Executes inline JavaScript code steps in Zapier to clean, transform, and map reservation payload data into standardized JSON structures.
  • AI Web Automation (Skyvern Layer):
    • Passes formatted payload to custom workflows in Skyvern.
    • Dynamically navigates and inputs reservation details into destination RMS targets (OpenTable or SevenRooms).
  • Feedback Loop & Exception Handling:
    • Skyvern returns completion status (Finished / Failed) back to Zapier.
    • Slack Notifications: Sends real-time team alerts to dedicated Slack channels for immediate visibility on processing outcomes.
    • Airtable Auditing: Automatically logs reservation status into designated Successful or Failed tracking tables so team members know exactly which reservations require manual review or manual intervention.

Tech Stack

Zapier • JavaScript • Skyvern (AI Web Automation) • Airtable • OpenTable API/UI • SevenRooms API/UI • Slack Webhooks

Key Impact

  • Operational Efficiency: Eliminates repetitive manual data entry across multiple RMS dashboards.
  • Error Prevention & Rapid Recovery: Instantly flags failed entries in Slack and routes them to dedicated Airtable review queues for fast human fallback.
  • Scalability: Allows Tablz to seamlessly handle higher reservation volumes without scaling manual operations headcount.
